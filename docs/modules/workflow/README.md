# 工作流引擎

**最后更新：** 2026-06-24
**状态：** ✅ 已完成

## 一句话职责

编排长任务的五个阶段，管理阶段间门控、文档产出、恢复节点和预审机制。

## 业务背景

长任务在会话间丢失上下文时会失败。下一个智能体从零开始，做出不一致的决策，重复已回答的问题。工作流引擎通过强制执行五个离散阶段——每个阶段有明确定义的输入、输出和门控条件——确保跨会话的连续性和一致性。

## 核心流程

### 任务生命周期

| 步骤 | 描述 | 代码 |
|------|------|------|
| 1 | 用户描述任务 → 第一阶段逐个提问、记录发现 | [SKILL.md:108-164](../../../SKILL.md#L108) |
| 2 | 架构师评估体量 → 创建 ARCHITECTURE.md | [SKILL.md:166-222](../../../SKILL.md#L166) |
| 3 | 文档记录员创建模块文档 → 定义工作包 | [SKILL.md:224-258](../../../SKILL.md#L224) |
| 4 | 实施者按工作包执行 → 更新代码指针 | [SKILL.md:260-281](../../../SKILL.md#L260) |
| 5 | 多专家独立审查 → 修复阻断项 | [SKILL.md:283-311](../../../SKILL.md#L283) |

### 门控流程

每个阶段完成后触发预审 → 修正 → 用户确认 → 进入下一阶段。

## 业务规则

| 规则 | 触发条件 | 代码 |
|------|---------|------|
| 单问规则 | 第一阶段每次只问一个问题 | [SKILL.md:113](../../../SKILL.md#L113) |
| 门控不可跳过 | 每个阶段结束必须获得用户批准 | [SKILL.md:161](../../../SKILL.md#L161) |
| 扁平快速通道 | 体量评估为小时跳过架构分层 | [SKILL.md:201](../../../SKILL.md#L201) |
| 动态调整 | 执行中发现分层不足可升级层数 | [SKILL.md:203](../../../SKILL.md#L203) |
| 文档同步门控 | 工作包完成=文档已更新，否则不可标记完成 | [SKILL.md:272](../../../SKILL.md#L272) |
