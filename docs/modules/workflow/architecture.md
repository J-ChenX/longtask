## 接口详情

### 阶段的抽象接口

```markdown
每个阶段：
  - 角色：{谁执行}
  - 输入：{从哪个阶段/文档来}
  - 输出：{产出什么文档}
  - 预审步骤：{Agent 预审清单}
  - 阶段门控：{用户确认条件}
  - 无人值守：{暂停时的状态标记位置}
```

## 数据流

```
用户描述
    │
    ▼
第一阶段：发现 ──→ _INDEX.md (发现部分) ──→ 预审 ──→ 用户批准
    │
    ▼
第二阶段：架构分层 ──→ ARCHITECTURE.md ──→ 预审 ──→ 用户批准
    │
    ▼
第三阶段：文档化 ──→ modules/{name}/*.md ──→ 预审 ──→ 用户批准
    │
    ▼
第四阶段：执行 ──→ 代码 + 更新代码指针 ──→ (第五阶段审查)
    │
    ▼
第五阶段：审查 ──→ review-log.md ──→ 用户批准
```
