**最后更新：** 2026-05-15
**状态：** 🔄 血肉进行中

## 一句话职责

定义多专家角色、审查协议、预审 Agent 和文档记录员——为工作流的质量和正确性提供人力（AI）保障。

## 业务背景

不同专家从不同领域视角独立审查，避免回音室效应。预审 Agent 在阶段门控前以干净视角检查上下文污染。文档记录员负责模块边界判定和文档生命周期管理。

## 核心流程

### 审查协议

| 步骤 | 描述 | 代码 |
|------|------|------|
| 1 | 依次运行每位专家，完全进入该角色 | [expert-roles.md:9-18](expert-roles.md#L9) |
| 2 | 从该角度阅读代码和文档 | 同上 |
| 3 | 记录发现（阻断/文档过时/警告/建议） | [expert-roles.md:168-177](expert-roles.md#L168) |
| 4 | 所有专家完成后汇总，标记跨专家分歧 | [expert-roles.md:15-18](expert-roles.md#L15) |

## 业务规则

| 规则 | 触发条件 | 代码 |
|------|---------|------|
| 独立审查 | 每位专家在分享前独立得出结论 | [expert-roles.md:7](expert-roles.md#L7) |
| 文档过时=阻断 | 过时文档严重性与损坏测试相同 | [expert-roles.md:175](expert-roles.md#L175) |
| 预审只跑一轮 | Agent 审→主 LLM 修→提交，不循环 | [expert-roles.md:42-43](expert-roles.md#L42) |
| 争议优先信主 LLM | Agent 争议标注供用户裁决 | [expert-roles.md:46](expert-roles.md#L46) |
| 模块判定三原则 | 人类直觉 + LLM 检索 + 内聚性 | [expert-roles.md:72-75](expert-roles.md#L72) |

## 严重性级别

| 级别 | 含义 | 是否阻断 |
|------|------|---------|
| 阻断 | 代码错误、缺失或危险 | 是 |
| 文档过时 | 文档与代码矛盾 | **是** |
| 警告 | 应改进但不紧急 | 否 |
| 建议 | 可选增强 | 否 |
